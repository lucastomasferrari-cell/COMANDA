Floors ​

The Floors module allows you to manage dining floors in each branch. Floors are used to organize tables and seating areas within the restaurant, making it easier to manage zones in the POS system.

Index Page ​

The Floors index page lists all available floors in a tabular view.

Filters ​

  • Branch – filter by branch to show floors assigned to that branch.
  • Activation – filter by status (Active / Inactive).
  • From / To – filter by creation date range.

Actions ​

  • Create Floor – add a new floor for a specific branch.
  • Row Actions
    • Edit Floor – update floor details.
    • Delete Floor – remove a floor (cannot be undone).
  • Bulk Actions
    • Delete Floors – delete multiple floors at once.

CRUD Attributes ​

AttributeDescriptionRequired
NameFloor name (translatable)Yes
BranchBranch to which this floor belongsYes
ActiveEnable/disable floor visibilityNo

Create Page ​

When creating a floor:

  • Name: Enter the floor name (required, translatable).
  • Branch: Select the branch where the floor will be created (required).
  • Active: Choose if the floor is immediately visible .

Example Use Case:

  • You may create Floor 1 and Floor 2 for the "Downtown Branch".
  • Assign tables later under each floor to organize seating zones.

Edit Page ​

When editing a floor:

  • You can update the Name and Activation Status.
  • The Branch cannot be changed after creation (to maintain data consistency).
  • Deactivating a floor will hide all tables under it in the POS system but does not delete them.

Example Use Case:

  • If a section of the restaurant is closed for renovation, mark the floor as Inactive.
  • Reactivate when the section is available again.

Notes ​

  • Floors are directly tied to Branches – you must assign each floor to one branch.
  • Deleting a floor also removes all tables created under that floor.
  • It’s recommended to disable a floor instead of deleting if you plan to reuse it later.
