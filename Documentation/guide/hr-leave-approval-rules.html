Leave Approval Rules ​

The Leave Approval Rules section defines conditional approval routing and auto-approval thresholds for leave requests.

Rules are evaluated by priority (higher first) and can include condition logic and approver chains.

Leave Approval Rules Index Page ​

The index page lists all approval rules by leave type and branch.

Available Filters ​

FilterDescription
BranchFilters rules by branch (visible when user is not branch-assigned).
Leave TypeFilters by leave type.
FromStart date for created-at filtering.
ToEnd date for created-at filtering.

Notes:

  • This table has no search box.

Available Actions ​

  • Create: Opens rule creation page.
  • Edit: Updates an existing rule.
  • Delete: Deletes one rule after confirmation.
  • Bulk Delete: Deletes selected rules after confirmation.

Create Leave Approval Rule Page ​

Use this page to define auto-approval thresholds, conditional matching rules, and approver chain.

Leave Approval Rule Information ​

FieldWhat it does
BranchRule scope branch (shown on create for non branch-assigned users).
Leave TypeLeave type this rule applies to.
Max Days Auto-ApproveAuto-approves day-based requests up to this threshold (shown for Day/Both units).
Max Minutes Auto-ApproveAuto-approves hour-based requests up to this threshold (shown for Hour/Both units).
PriorityRule order priority; higher priority is evaluated first.
Requires AttachmentForces attachments for matching requests even if leave type does not require one.
ActiveEnables/disables rule participation in workflow resolution.

Conditions & Approvers ​

Match Mode ​

  • All conditions: all condition rows must pass.
  • Any condition: at least one condition row must pass.

Condition Builder ​

Available condition fields:

  • Employee
  • Department
  • Designation
  • Effective Days (day-capable leave types)
  • Effective Minutes (hour-capable leave types)
  • Tenure (Months)

Available operators:

  • Equals
  • Not Equals
  • In
  • Not In
  • Greater Than Or Equal
  • Less Than Or Equal
  • Greater Than
  • Less Than

Behavior:

  • In / Not In accept multiple values.
  • Numeric fields (effective_days, effective_minutes, tenure_months) require numeric values.
  • Invalid field/unit combinations are blocked:
    • effective_days cannot be used when leave type unit is Hour.
    • effective_minutes cannot be used when leave type unit is Day.

Approver Chain ​

Approver types:

  • Direct Manager
  • Role
  • Specific User

How it works:

  • Approvers are processed in the order listed in the form.
  • Each approver row creates one approval level.
  • If manager is selected but no manager is resolvable, system falls back to available admin roles by priority.

Edit Leave Approval Rule Page ​

Use this page to update conditions, approvers, thresholds, or status.

The edit page uses the same structure and rules as create.

Validation Rules ​

  • Leave Type: required.
  • Max Days Auto-Approve: optional numeric 0..999999.
  • Max Minutes Auto-Approve: optional integer 0..999999.
  • Priority: required integer 1..1000.
  • Requires Attachment: required boolean.
  • Active: required boolean.
  • Condition values are required when field/operator is defined.
  • Role/User approver entries require valid values.
