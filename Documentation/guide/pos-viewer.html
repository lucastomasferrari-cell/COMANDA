POS Viewer ​

The POS Viewer is the core interface for handling restaurant orders, managing tables, processing payments, and monitoring activities.
It provides a streamlined workflow for waiters, cashiers, and managers to handle dine-in, takeaway, and pick-up orders.

Main Page ​

The Main Page is divided into the Top Bar, Content (Left Side), and Content (Right Side).

Top Bar ​

  • Left Side

    • Back to Dashboard: Return to the admin dashboard.
    • Search by Product Name: Instantly filter products by typing product keywords.

  • Right Side

    • Table Viewer Button: Opens the Table Viewer to manage tables.
    • Cash Movements Button: Opens the cash movement window to track in/out transactions.
    • Active Orders Button: Displays all active orders in the system.
    • Cart & Items Counter: Shows the number of items in the cart and allows clearing all items.
    • POS Register Dropdown: Select which register (e.g., “Bakery Counter”) to use for the session.

Content ​

  • Left Side

    • Categories with Subcategories: Displays product categories (Appetizers, Burgers, Drinks, etc.) and their subcategories.
    • Products List: Shows product cards with images, prices, discounts, and “Add” button for quick selection.

  • Right Side

    • Order Type Selection
      • Takeaway
      • Pick-up
      • Dine-in (if a table is selected, this is automatically applied and other types are hidden).
    • Added Products
      • Lists products added to the cart.
      • Shows modifiers (e.g., sauce, toppings) and options.
      • Allows editing/removing items.
    • Order Summary
      • Displays subtotal, applied taxes, and total.
    • Payment Section
      • Select payment method (Cash, Card, or Mixed) if order type is not dine-in.
    • Additional Options
      • Add notes to the order (optional).
      • Guest count field (default = 1).
    • Place Order
      • Confirms and submits the order.
      • If already paid, a dialog appears to print the receipt.

Order Printing ​

From POS order actions, users with print permission can open the Print Order Documents dialog.

  • Branch: Shown only when the user is not restricted to a single branch.
  • POS Register: Required for direct printing bill, invoice, and waiter documents.
  • Preview Print: Opens a browser print preview for the selected document.
  • Direct Print: Sends the document to the branch’s active print agent through Qintrix.

Available document types ​

  • Bill: Available from the order print dialog.
  • Invoice: Available only after the order is paid.
  • Waiter copy: Available from the order print dialog.
  • Kitchen tickets: Added per kitchen when the order contains matching products and the order is not completed.

Direct-print requirements ​

  • The branch must have an active Print Agent configured with API key, host, and port.
  • The destination printer must have a valid Qintrix ID.
  • A register must be selected for non-kitchen direct-print actions.

Table Viewer Page ​

The Table Viewer provides a visual layout of all tables filtered by floor, zone, and status.

  • Left Side (Filters)

    • Status Filters: Available, Occupied, Reserved, Cleaning, Merged.
    • Floor Filters: All floors or a specific floor.
    • Zone Filters: All zones, or specific zones (e.g Outdoor, Indoor, Terrace, Quiet Zone).

  • Right Side (Tables)

    • Displays active tables based on filters.
    • Clicking a table opens Table Details.

Table Details Page ​

When a table is selected, a detailed view appears with its capacity, floor, zone, waiter, and orders.



Merge Details ​

  • Displays merge information (if the table is merged with others).
  • Merge types:
    • Order/Billing Merge: Shows all orders for merged tables.
    • Capacity Merge: Shows only orders for that specific table.

Orders Details ​

  • Displays all orders linked to the table.
  • Shows:
    • Order number
    • Linked table(s)
    • Time
    • Total
    • Order status (Confirmed, Preparing, etc.)
    • Payment status (Paid/Unpaid)

Table Statuses & Actions ​

  • Available

    • Merge Table
    • Create Order
    • Assign Waiter
    • Split Merge (if merged)

  • Cleaning

    • Assign Waiter
    • Make Available
    • Split Merge (if merged)

  • Occupied

    • Update Order Status
    • Show Order Details
    • Cancel Order (if available)
    • Refund Order (if available)
    • Mark as Paid (if merged type billing → pay all linked orders together)

Cash Movement Page ​

The Cash Movement page records money in/out activities.

Actions ​

  • In: Register cash received (e.g., deposits).
  • Out: Register cash going out.
  • Attributes:
    • Amount
    • Reference (optional)
    • Notes (optional)

Active Orders Page ​

Displays all ongoing orders (excluding dine-in) with filters.

  • Left Side (Filters)

    • Order Types: Takeaway, Pick-up
    • Status: Pending, Confirmed, Preparing, Ready, Completed
    • Payment Status: Paid, Unpaid

  • Right Side (Order Cards)

    • Each card shows:
      • Order number
      • Order type
      • Time
      • Total
      • Order status
      • Payment status
    • Actions:
      • Update Order Status
      • Show Order Details
      • Cancel Order (if available)
      • Refund Order (if available)
      • Mark as Paid

Payment Handling ​

  • Orders can be settled by:
    • Cash
    • Card
    • Mixed (split between cash and card)
  • Once marked Paid:
    • Order is completed if order is ready/served.
    • Receipt printing option is displayed.

Summary ​

The POS Viewer connects all essential restaurant operations:

  • Manage tables visually.
  • Place and edit orders.
  • Handle cash transactions.
  • Track active orders with filters.
  • Process payments with multiple methods.
  • Ensure smooth coordination between waiters, cashiers, and managers.

This ensures efficient order management, reduces errors, and keeps financials accurate.
