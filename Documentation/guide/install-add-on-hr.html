HR Add-on Installation Guide ​

This guide explains how to install the forkiva-hr add-on in this repository in two deployment styles:

  1. Frontend and backend are separate (frontend/ + backend/)
  2. Frontend and backend are combined (forkiva/)

The add-on package is located at:

  • forkiva-hr/backend (Laravel module)
  • forkiva-hr/frontend (UI files)

1) Quick structure (client view) ​

  • backend/ = Laravel API app with modular architecture (Modules/*)
  • frontend/ = Vue SPA source code
  • forkiva/ = merged production-ready app (Laravel + built frontend in public/)

The HR add-on follows the same backend module model, and the frontend part is installed as a module folder under frontend/src/modules/hr.

2) Method A: Install when frontend and backend are separate ​

Run all commands from project root: forkiva

A. Install backend part (backend/) ​

  1. Copy module files:
bash
mkdir -p backend/Modules/Hr
cp -R forkiva-hr/backend/. backend/Modules/Hr/
  1. Enable module (choose one):
bash
cd backend
php artisan module:enable Hr

Or manually in backend/modules_statuses.json add:

json
"Hr": true
  1. Refresh autoload + cache:
bash
composer dump-autoload
php artisan optimize:clear
  1. Run upgrade command:
bash
php artisan forkiva:upgrade

B. Install frontend part (frontend/) ​

This frontend is module-based and auto-loads module manifests from:

  • frontend/src/modules/**/manifest.ts

So install HR as a full frontend module under src/modules/hr.

  1. Copy all HR frontend files:
bash
mkdir -p frontend/src/modules/hr
cp -R forkiva-hr/frontend/. frontend/src/modules/hr/
  1. No manual route/sidebar registration is needed.

The HR module already includes:

  • manifest.ts
  • routes/admin.routes.ts
  • sidebars/admin.sidebar.ts

These are auto-collected by the app bootstrap.

  1. Build frontend:
bash
cd frontend
yarn install
yarn build

C. Verify (separate mode) ​

  1. Backend route exists:
bash
cd backend
php artisan route:list | grep -i hr
  1. Start apps and open admin UI:
  • confirm sidebar has HR entry
  • confirm departments list/create/edit pages open

3) Method B: Install when frontend and backend are combined (forkiva/) ​

For this repository, best practice is to install and test in separate mode first, then apply the same module/frontend wiring to the combined app.

A. Install backend in merged app ​

bash
mkdir -p forkiva/Modules/Hr
cp -R forkiva-hr/backend/. forkiva/Modules/Hr/
cd forkiva
php artisan module:enable Hr
composer dump-autoload
php artisan optimize:clear
php artisan forkiva:upgrade

B. Install frontend in merged app ​

Do not edit forkiva/public/assets/* manually. Those are compiled files.

Use this flow:

  1. Ensure the HR frontend module exists in your source app as src/modules/hr (same structure as Method A, Section 2-B).
  2. Build frontend with your normal build process, then publish/update compiled assets in forkiva/public/ according to your deployment pipeline.

C. Verify (combined mode) ​

bash
cd forkiva
php artisan route:list | grep -i hr
php artisan optimize:clear

Then open merged app and validate:

  • HR sidebar/menu exists
  • Department CRUD pages load
  • API requests return success (not 404/401)

4) Common issues ​

  1. Class "Modules\\Hr\\..." not found
  • run composer dump-autoload
  • ensure folder is exactly Modules/Hr
  1. Module not active
  • check modules_statuses.json includes "Hr": true
  1. Frontend module not loading
  • confirm frontend/src/modules/hr/manifest.ts exists
  • ensure no syntax/build errors in frontend/src/modules/hr/**
  • run frontend build again
  1. Sidebar item not visible
  • user may lack permission (admin.departments.index)
  • verify backend permissions/seeding for HR module

5) Recommended release sequence ​

  1. Install backend + frontend in separate mode
  2. Test locally
  3. Build and publish merged artifacts using your standard release pipeline
  4. Deploy merged forkiva/
