Orders ​

In Forkiva, Orders represent the core activity of the restaurant POS system.
Each request (order) captures the entire transaction lifecycle — from the moment a customer places an order until it is billed, served, refunded, or cancelled.

View Orders ​

The Orders page provides a centralized view of all restaurant orders.
Admins and staff can view, filter, and manage orders depending on their assigned branch and granted permissions.

Filters ​

The order list can be refined using the following filters:

  • Branch – Visible only if the user is not assigned to a specific branch. Allows selecting one branche to filter orders.
  • Status – Filter orders by their current status.
  • Order Type – Filter by type of service.
  • Payment Status – Filter orders by payment state (Paid / Unpaid).
  • Created From – Show orders created on or after a specific date.
  • Created To – Show orders created on or before a specific date.

Order Table ​

Each row in the orders table displays:

  • Reference No – Unique order reference code.
  • Branch – The branch where the order was created.
  • Type – Order type
  • Status – Current status of the order.
  • Payment Status – Paid or Unpaid.
  • Total – Order total amount in branch currency.
  • Created At – Date and time when the order was placed.
  • Actions – Available actions depending on permissions.

Row Actions ​

Row-level actions are permission-based and conditionally displayed:

  • Show Order Details

    • Permission required: admin.orders.show
    • Opens the full details of the order.

  • Cancel Order

    • Permission required: admin.orders.cancel
    • Available only if the order status allows cancellation.
    • Cancels the order and updates stock/cash flow accordingly.

  • Refund Order

    • Permission required: admin.orders.refund
    • Available only if the order status allows refund.
    • Processes a refund and adjusts stock/cash movement.

Notes ​

  • If the logged-in user is assigned to a specific branch, only orders from that branch are visible, and amounts are displayed in that branch’s currency.
  • If the user is not branch-specific, orders across all branches are shown and totals are displayed in the default system currency.
  • If the user is not branch-specific, orders across all branches are shown. Totals are displayed in the respective branch currency, and on hover the price is shown in the system default currency.

Show Order Details ​

The Show Order Details page provides a complete breakdown of an order, including its general information, ordered products, payment details, and status logs.
This page helps administrators, managers, and authorized staff track the full lifecycle of an order.

Order Information ​

Displays key metadata about the order:

  • Reference No & Order Number – Unique identifiers for the order.
  • Branch – The branch where the order was placed.
  • Status – Current status of the order (e.g., Pending, Confirmed, Preparing, Completed).
  • Type – Order type (e.g., Dine-In, Takeaway, Pick-up).
  • Payment Status – Shows whether the order is Paid or Unpaid.
  • Guest Count – Number of guests .
  • Created By – User who created the order.
  • Cashier – Staff member who processed the order.
  • POS Register – The POS register used.
  • POS Session ID – The POS session associated with the order.
  • Created At / Updated At – Timestamps for when the order was placed and last updated.

Order Summary ​

Provides the financial breakdown of the order:

  • Currency & Rate – Currency applied (branch currency by default).
  • Subtotal – Total amount before taxes.
  • VAT/Taxes – Calculated tax amount.
  • Total – Final total including taxes.

Products ​

List of items included in the order:

  • Product Name – Ordered product(s).
  • Options/Modifiers – Product options (e.g., size, ice level, toppings).
  • Unit Price – Price per unit.
  • Quantity – Number of units ordered.
  • Subtotal – Total cost before tax.
  • Tax – Applied tax for the product.
  • Total – Final cost including tax.

Payments ​

Details of payment methods used for the order:

  • Payment Method(s) – e.g., Cash, Card, Mixed.
  • Amount Paid – Total amount per method.

Notes ​

  • Any additional notes added by the cashier, waiter, or system regarding the order.

Status Logs ​

Chronological history of order status changes:

  • Status – Shows each change (e.g., Pending → Confirmed → Preparing).
  • Changed By – Identifies the user/system that updated the status.
  • Reason – Reason for status change (if provided).
  • Note – Any additional comments tied to the change.
  • Changed At – Timestamp of the status update.

Notes ​

  • If the user is branch-specific, all amounts are displayed in the branch’s currency.
  • If the user is not branch-specific, totals are displayed in the branch’s currency, with the option to hover over prices to see them in the system default currency.
