Gift Cards ​

The Gift Cards module allows your team to issue, sell, track, redeem, and report on stored-value gift cards across branches.

Gift cards are useful when you want to:

  • Sell prepaid value to customers
  • Redeem gift cards as a payment method in orders
  • Recharge an existing card
  • Refund an order back to a gift card
  • Track outstanding balances and liability
  • Review all card movements through reports

The module is organized into three main areas:

  • Cards: Manage individual gift cards
  • Transactions: Review all gift card movements
  • Reports: Analyze sales, redemption, balances, expiry, and liability

Gift Card Concepts ​

Before using the module, it helps to understand the main fields:

FieldDescription
Card CodeThe unique gift card number used during lookup and payment.
Initial BalanceThe original stored amount when the card was created or sold.
Current BalanceThe remaining usable amount on the card.
CurrencyThe card currency. This is assigned automatically from the selected branch, or from the default currency if no branch is selected.
BranchThe branch assigned to the card. If no branch is selected, the card becomes global.
ScopeSet automatically by the system. Branch when a branch is selected, Global when no branch is selected.
StatusUsually Active, Used, Expired, or Disabled.
Expiry DateOptional date after which the card can no longer be used.
CustomerOptional customer linked to the gift card.
BatchA batch used for bulk-generated cards. This is set by the system, not manually in the card form.

Cards ​

The Cards page shows all gift cards in the system.

From this page, users can:

  • View all existing cards
  • Search by card code, UUID, or customer
  • Filter by branch, currency, scope, status, and date
  • Open a card to view its full details and transaction history
  • Create a new card manually
  • Edit card details

Card List Columns ​

Typical columns include:

  • Card code
  • Branch
  • Customer
  • Initial balance
  • Current balance
  • Currency
  • Scope
  • Status
  • Expiry date
  • Created at

Filters ​

You can use filters such as:

FilterDescription
BranchShow only cards for a selected branch
CurrencyShow cards for a selected currency
ScopeFilter by Branch or Global
StatusFilter by Active, Used, Expired, or Disabled
From / ToFilter by creation date

Create Gift Card ​

The Create Gift Card page is used when you want to create a card directly from the admin panel.

Main Rules ​

  • Code is optional. If left empty, the system generates it automatically.
  • Currency is not entered manually.
    • If you select a branch, the card currency is taken from that branch.
    • If branch is empty, the card currency is taken from the default system currency.
  • Scope is not selected manually.
    • With branch: Branch
    • Without branch: Global
  • Batch cannot be assigned manually from the card form.

Recommended Use Cases ​

  • Create a one-off card for a customer
  • Create a manual administrative card
  • Create a replacement card when needed

Gift Card Details Page ​

The Gift Card Details page gives a full overview of one card.

It includes:

  • Card code
  • UUID
  • Branch
  • Customer
  • Initial balance
  • Current balance
  • Currency
  • Scope
  • Status
  • Expiry date
  • Notes
  • Created and updated timestamps
  • Related batch, if available

The page also includes a Transactions table showing all movements related to that card.

Transactions ​

The Transactions page lists all gift card ledger movements.

This page is useful for:

  • Auditing card activity
  • Reviewing redemptions and refunds
  • Tracking recharge history
  • Investigating balance changes

Transaction Types ​

TypeDescription
PurchaseInitial card sale or card issuance value
RedeemValue used to pay an order
RefundValue returned to a gift card from a refunded order
RechargeAdditional value added to a card
AdjustmentManual administrative balance change
ExpireSystem-created transaction when remaining balance expires

Transaction List Columns ​

Common columns include:

  • Transaction ID
  • Gift card
  • Type
  • Amount
  • Balance before
  • Balance after
  • Currency
  • Order
  • Branch
  • Created by
  • Transaction date

Transaction Details Page ​

Each transaction can be opened to view full details such as:

  • UUID
  • Gift card
  • Type
  • Amount
  • Balance before
  • Balance after
  • Order reference
  • Exchange rate, if applicable
  • Amount in order currency, if applicable
  • Branch
  • Created by
  • Notes
  • Transaction date
  • Created at

Card Batches ​

The Card Batches page is used when you need to generate multiple gift cards at once.

Typical Use Cases ​

  • Holiday campaign cards
  • Promotional cards for events
  • Bulk prepaid cards for corporate customers
  • Branch-specific seasonal campaigns

Batch Fields ​

FieldDescription
NameBatch name
PrefixOptional code prefix used during generation
QuantityNumber of cards to create
ValueInitial amount assigned to each generated card
BranchOptional branch
CurrencySet automatically from the branch, or from the default currency if branch is empty

Important Notes ​

  • Batch cards are created automatically when the batch is saved
  • The generated cards inherit batch value and currency
  • If the batch has a branch, generated cards are branch-scoped
  • If the batch has no branch, generated cards are global

POS Usage ​

Gift cards are also used operationally in POS workflows.

NOTE

Direct Sell Gift Card as a dedicated POS action is not currently available in the system. Gift cards are currently managed through the Gift Cards module, while POS supports gift card usage in payment and refund flows.

Redeem Gift Card ​

Gift cards can be used in the payment dialog as a payment method.

Typical flow:

  1. Choose Gift Card as payment method
  2. Enter or scan the card code
  3. The system validates the card
  4. The amount is deducted from the remaining balance
  5. The order is completed if payment is sufficient

Refund to Gift Card ​

During order refund, the refund amount can be returned to a gift card instead of cash or bank payment.

This creates a Refund transaction on the gift card and increases its available balance.

Reports ​

The Gift Cards module includes several dedicated reports.

Gift Card Sales Report ​

Shows all gift cards sold during a selected period.

Useful for:

  • Reviewing gift card sales activity
  • Tracking who sold cards
  • Comparing sales between branches

Gift Card Redemption Report ​

Shows all gift cards used in orders.

Useful for:

  • Tracking payment usage
  • Auditing redemptions
  • Reviewing converted amounts in different order currencies

Gift Card Outstanding Balance Report ​

Shows active cards and their remaining balances.

Useful for:

  • Identifying unused value
  • Monitoring cards that still hold customer funds

Gift Card Liability Report ​

Shows the financial liability of unused gift card balances.

Useful for:

  • Finance and accounting review
  • Understanding stored-value exposure by currency

Gift Card Expiry Report ​

Shows cards that are near expiry or already expired.

Useful for:

  • Operational follow-up
  • Customer service review
  • End-of-period expiry monitoring

Gift Card Transactions Report ​

Shows all transaction activity across gift cards.

Useful for:

  • Auditing
  • Investigation
  • Reviewing manual adjustments and expiry transactions

Gift Card Branch Performance Report ​

Compares gift card activity across branches.

Useful for:

  • Branch-level performance review
  • Comparing sold, redeemed, and outstanding amounts

Gift Card Batch Report ​

Shows statistics about generated batches.

Useful for:

  • Bulk issuance review
  • Campaign tracking
  • Measuring usage of generated cards

Best Practices ​

  • Use global cards when a card should be accepted across all branches
  • Use branch cards when the card should remain tied to one branch
  • Always set an expiry date only if your business policy requires it
  • Use batches for campaign-based or large-volume issuance
  • Review outstanding balance and liability reports regularly
  • Investigate adjustment transactions carefully because they affect stored value directly

Operational Notes ​

  • A card with no remaining balance may move to Used
  • Expired cards cannot be redeemed
  • Disabled cards cannot be used operationally
  • Currency is managed automatically by the system
  • Card codes can be entered manually by the cashier and are formatted to be easier to read and type

Example Workflow ​

  1. Create a batch for a marketing campaign or create a card manually
  2. Sell the card to a customer
  3. Customer uses the card later in POS payment
  4. The system creates ledger transactions automatically
  5. Review card history on the card details page
  6. Monitor balances, redemptions, expiry, and liability through reports
