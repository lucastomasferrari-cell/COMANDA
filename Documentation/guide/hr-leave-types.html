On this page

Leave Types ​

The Leave Types section defines leave categories and their behavior across leave requests, approvals, balances, attendance, and policy rules.

Leave Types Index Page ​

The index page lists leave types with code, unit, approval/attachment requirements, paid flag, activation status, and creation date.

Available Filters ​

FilterDescription
SearchSearches by leave type code or name.
UnitFilters by leave unit (Day, Hour, Both).
ActivationFilters by active or inactive leave types.
FromStart date for created-at filtering.
ToEnd date for created-at filtering.

Available Actions ​

  • Create: Opens the leave type creation form.
  • Edit: Opens the leave type edit form.
  • Delete: Deletes one leave type after confirmation.
  • Bulk Delete: Deletes selected leave types after confirmation.

Create Leave Type Page ​

Use this page to add a new leave type.

Leave Type Information Card ​

FieldWhat it does
CodeUnique internal code for the leave type (for example: ANNUAL, SICK).
Name (multi-language)End-user leave type name in each enabled language.
Description (multi-language)Optional explanatory text shown in HR context.
UnitDefines allowed request measurement: Day, Hour, or Both.

Rules & Behavior Card ​

OptionWhat it does
Deduct From BalanceIf enabled, approved leave consumes employee leave balance. If disabled, no accrual/carryover/limit logic can be applied in leave policies for this type.
Affects AttendanceMarks approved leave as attendance-impacting in attendance workflows.
Requires ApprovalSets default approval requirement for this leave type.
Requires AttachmentRequires supporting files when creating leave requests for this type.
PaidMarks whether leave is paid. Unpaid leave cannot use accrual in leave policies.
ActiveControls whether this leave type is available for use in forms and policy setup.

Unit Values ​

  • Day: Leave requests are measured in days only.
  • Hour: Leave requests are measured in hours/minutes only.
  • Both: Request can be submitted either as day-based or hour-based.

Validation Rules ​

  • Code: required, max 50 chars, unique.
  • Name: required, min 2 and max 255 chars per locale.
  • Description: optional, min 2 and max 1000 chars when entered.
  • Unit: required enum value (day, hour, both).
  • All behavior toggles (Requires Approval, Requires Attachment, Paid, Deduct From Balance, Affects Attendance, Active) are required boolean values.

Edit Leave Type Page ​

Use this page to modify an existing leave type.

The edit page uses the same structure and options as create.

How Leave Type Options Affect Other Sections ​

  • Requires Approval is the default approval behavior during leave workflow.
  • If a Leave Approval Rule exists for the same request scope, that rule overrides default leave-type approval behavior.
  • Active leave types are the ones available when creating leave policies and requests.
