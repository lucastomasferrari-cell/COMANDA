On this page

Leave Requests ​

The Leave Requests section manages the full lifecycle of leave requests: draft, submit, approval, and cancellation.

Leave Requests Index Page ​

The index page lists requests with period, unit, status, effective duration, and submission time.

Available Filters ​

FilterDescription
SearchSearches by employee name, leave type name, or request UUID.
BranchFilters by branch (visible when user is not branch-assigned).
EmployeeFilters by employee (depends on branch).
Leave TypeFilters by leave type.
UnitFilters by request unit (Day or Hour).
StatusFilters workflow status (Draft, Pending, Approved, Rejected, Cancelled).
FromStart date filter.
ToEnd date filter.

Available Actions ​

  • Create: Opens request creation form.
  • Show: Opens request details.
  • Edit: Available only for Draft requests.
  • Submit: Available only for Draft requests.
  • Cancel: Available for Pending or Approved requests (with confirmation).

Create Leave Request Page ​

Use this page to create a request draft.

Request Details ​

FieldWhat it does
BranchBranch context (shown on create for non branch-assigned users).
EmployeeEmployee who owns the request (locked on edit).
Leave TypeRequested leave type.
UnitRequest unit. For leave types with Both, user can choose Day or Hour; otherwise auto-locked.

Reason & Attachments ​

FieldWhat it does
ReasonOptional free-text reason (max 1000 chars).
AttachmentsOptional supporting files (required by some leave types/rules).

Accepted attachment types include PDF, images, Word, Excel, and CSV with max file size 5 MB per file.

Schedule ​

FieldWhat it does
Start DateRequest start date.
End DateRequest end date (for day-based requests).
Start TimeRequired for hour-based requests.
End TimeRequired for hour-based requests; must be after start time.

Edit Leave Request Page ​

Use this page to update a draft request.

Rules:

  • Only Draft requests are editable.
  • On hour unit, end date is forced to start date.

Leave Request Show Page ​

Use this page for full request review and audit trace.

Visible Details ​

  • Core request fields and calculated effective duration.
  • Attachments (preview/download when applicable).
  • Approval timeline and approver comments.
  • Logs with action and performer.
  • Submission and approval timestamps.

Validation and Workflow Rules ​

Basic Validation ​

  • Employee: required and must exist.
  • Leave Type: required and must exist.
  • Unit: required (day or hour).
  • Start/End Date: required; end date must be on/after start date.
  • Hourly requests:
    • start and end times are required
    • times must be valid format (HH:mm or hh:mm AM/PM)
    • end time must be after start time
    • start and end date must be same day

Submission Rules ​

  • Only Draft can be submitted.
  • Request is rejected when:
    • no eligible working days/hours found
    • hourly time does not overlap assigned shift
    • overlaps an existing leave request
    • required attachments are missing
    • policy max per request is exceeded
    • leave balance is insufficient (or exceeds allowed negative limit)

Cancellation Rules ​

  • Only Pending or Approved can be cancelled.
  • Cancelling an approved request creates reversal ledger transactions.
