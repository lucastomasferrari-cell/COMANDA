Printers ​

The Printers module is used to register the printable destinations that Forkiva can target through Qintrix. Each printer record belongs to a branch and stores the Qintrix printer identifier plus the receipt paper size.

Printers work together with Print Agents:

  • The Print Agent tells Forkiva which Qintrix service endpoint to call.
  • The Printer tells Forkiva which Qintrix printer ID should receive the job.

If a printer does not have a valid Qintrix ID, Forkiva cannot send direct print jobs to it.

Index Page ​

The index lists all configured printers:

  • Name – Friendly printer label.
  • Qintrix ID – Printer identifier used when creating Qintrix jobs.
  • Branch – Branch that owns the printer.
  • Activation – Whether the printer is active.
  • Created at / Updated at – Audit timestamps.
  • Actions – Edit or delete the printer.

Features ​

  • Search box for quick filtering.
  • Pagination with entry count selector.
  • Clear visibility of the Qintrix ID assigned to each printer.

➕ Create Printer ​

Use this form to create a printer that can be selected by Forkiva print flows.

Printer Information ​

  • Name – Printer display name. This field supports translations.
  • Branch – Target branch. This appears for users who are not restricted to a single branch.
  • Active – Enables the printer for printing flows.

Connection Options ​

  • Qintrix ID – Required printer identifier from your Qintrix setup.
  • Paper Size – Required paper width used when rendering print content.

Currently available paper sizes:

  • 58mm
  • 80mm

Edit Printer ​

The edit page lets you update the saved printer configuration:

  • Name
  • Active
  • Qintrix ID
  • Paper Size

Branch is fixed after creation.

How printers are used ​

Forkiva selects printers from different places depending on the print type:

  • Bill printer – Used for bill prints from POS.
  • Invoice printer – Used for invoice prints, typically after payment.
  • Waiter printer – Used for waiter copies.
  • Kitchen printers – Used for kitchen tickets when products match the kitchen mapping.

For direct printing to work, the selected printer must have a valid Qintrix ID.

Requirements for direct printing ​

Before direct printing works correctly, make sure all of the following are true:

  • The branch has an active Print Agent configured with API key, host, and port.
  • The printer is active.
  • The printer has a valid Qintrix ID.
  • The printer uses the correct paper size.
  • The related POS register or kitchen mapping is configured to use that printer.

If you have not configured the print agent yet, see Set Up Print Agent.

📌 Notes ​

  • Qintrix IDs are required for direct printing.
  • Preview printing can still work even when direct printing is unavailable.
  • Use clear printer names so staff can identify the correct destination during setup and troubleshooting.
