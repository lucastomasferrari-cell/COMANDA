Dashboard ​

The Dashboard is the first page an administrator sees after logging into the Forkiva Restaurant POS System.
It provides a real-time overview of restaurant performance, sales, inventory alerts, and key metrics—allowing managers to make quick, informed decisions.

Page Layout ​

The dashboard is divided into the following sections:

1. Key Metrics (Top Cards) ​

These highlight the most important restaurant statistics:

  • Total Sales – Displays the cumulative sales amount for all time.
  • Total Orders – Shows the number of orders placed, excluding Cancelled, Refunded, and Merged orders.
  • Total Active Orders – Current open/active orders in progress.
  • Average Order Value – Average value of all completed orders.
  • Total Users – Number of registered staff in the system.
  • Total Menus – Total number of menus.
  • Total Products – Count of all products across active menus.
  • Total Categories – Number of product categories available across active menus.

Each card uses visual icons and color-coded backgrounds to make insights quick to understand.

2. Hourly Sales Trend ​

  • Type: Line Chart
  • Description: Displays sales activity across the day, broken down by hour.
  • Use Case: Helps identify peak hours of business to optimize staff scheduling and promotions.

3. Sales Analytics ​

  • Type: Line Chart
  • Filters: Weekly, Monthly
  • Description: Shows sales performance over a given week or month.
  • Use Case: Compare sales patterns over time and track growth.

4. Top Selling Products ​

  • Type: List with product thumbnails and sales amounts.
  • Filters: All Time, Today, This Week, This Month, This Year
  • Description: Displays the best-performing products by sales quantity and amount.
  • Use Case: Track customer preferences and stock popular items.

5. Branch-wise Sales Comparison ​

  • Type: Bar Chart
  • Filters: All Time, Today, This Week, This Month, This Year
  • Description: Compares total sales between different restaurant branches.
  • Use Case: Identify top-performing locations and underperforming branches.

6. Best Performing Branches ​

  • Type: List with branch names and sales values.
  • Filters: All Time, Today, This Week, This Month, This Year
  • Description: Ranks branches based on total sales.
  • Use Case: Benchmark branch performance and incentivize staff.

7. Order Type Distribution ​

  • Type: Pie/Donut Chart
  • Filters: All Time, Today, This Week, This Month, This Year
  • Order Types: Dine-In, Takeaway, Pick-Up
  • Description: Shows the total orders by type.
  • Use Case: Understand customer preferences for service channels.

8. Order Total by Status ​

  • Type: Pie/Donut Chart
  • Filters: All Time, Today, This Week, This Month, This Year
  • Statuses: Pending, Confirmed, Preparing, Ready, Served, Completed, Cancelled, Refunded, Merged
  • Description: Breakdown of all orders by their current or final status.
  • Use Case: Identify fulfillment efficiency and canceled order patterns.

9. Low Stock Alerts ​

  • Type: List with ingredient names, stock quantity, and alerts.
  • Description: Identifies ingredients nearing depletion or finished.
  • Use Case: Prevent stockouts by prompting timely restocking.

10. Cash Movements Overview ​

  • Type: Pie/Donut Chart
  • Filters: All Time, Today, This Week, This Month, This Year
  • Directions: In, Out, Adjust
  • Description: Summarizes all cash flow movements across POS registers.
  • Use Case: Monitor cash drawer activity and prevent discrepancies.

11. Payments Overview ​

  • Type: Pie/Donut Chart
  • Filters: All Time, Today, This Week, This Month, This Year
  • Methods: Cash, Card
  • Description: Shows the distribution of payment methods used.
  • Use Case: Plan cash handling and evaluate customer payment preferences.

Notes on Data Visibility ​

  • All data displayed on the dashboard is contextual to the logged-in user.
  • If the user is not affiliated with a specific branch, the dashboard shows all branch data and applies the default system currency.
  • If the user is assigned to a specific branch, the dashboard shows only that branch’s data and uses the branch’s local currency.

Summary ​

The Dashboard serves as a control center for restaurant operations.
It provides managers and owners with instant insights into sales, branch performance, order statuses, payment methods, and inventory alerts—ensuring better decision-making and operational efficiency.
